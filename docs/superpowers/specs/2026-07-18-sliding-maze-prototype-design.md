# 滑行迷宫玩法原型设计

## 目标

在 Unity 2022.3 中制作一个 3D 俯视角滑行迷宫原型。玩家预先编排固定数量的方向指令；每条指令使角色沿对应方向持续移动，直到撞上障碍或地图边界。全部指令执行完成后，角色必须恰好停在终点才算通关。

首版只制作一个固定关卡并使用基础几何体占位，但关卡内容通过场景组件配置，方便后续复制场景并手工制作更多关卡。

## 范围

### 包含

- 一个可游玩的 3D 俯视角关卡。
- 场景内配置地图尺寸、障碍、起点、终点和指令数量 X。
- 屏幕方向按钮以及 WASD、方向键两种输入方式。
- 固定长度的指令队列。
- 执行、清空、成功、失败和重置流程。
- 基础方块、颜色、文字和移动动画。
- 核心网格逻辑测试及场景运行验证。

### 不包含

- 自动计算最小解。
- 随机关卡生成。
- 游戏内关卡编辑器。
- 单条指令撤销、替换或拖动排序。
- 参考图中的具体美术、特效、音效和教程界面复刻。

## 核心规则

1. 每关手动配置指令数量 X。
2. 玩家必须填满 X 条指令才能开始执行。
3. 玩家可通过屏幕四方向按钮、WASD 或方向键添加指令。
4. 填满后，玩家点击“执行”按钮或按空格键开始。
5. 执行前只能清空整组指令；不支持撤销或替换单条指令。
6. 每条指令使角色逐格滑行，直到下一格是障碍或地图边界。
7. 如果指令方向紧贴障碍，角色不移动，但该指令仍被消耗。
8. 前一条指令完全停止后，才执行下一条。
9. 滑行途中经过终点不触发成功。
10. 全部 X 条指令执行完毕且角色最终停在终点时通关。
11. 最终位置不是终点时提示失败；提示结束后角色回到起点、清空队列并重新允许输入。
12. 执行期间锁定添加、清空和重复执行操作。

## 架构

### LevelController

保存和验证关卡配置：

- 地图宽度和高度。
- 单格世界尺寸。
- 指令数量 X。
- 玩家、起点、终点引用。
- 障碍物父节点；启动时扫描其子节点上的 `Obstacle` 组件。
- 移动速度和失败提示时长。

启动时把场景对象的位置换算成整数网格坐标，并建立障碍坐标集合。网格规则不依赖物理碰撞结果。

### PlayerMover

- 持有角色当前整数网格坐标。
- 根据方向计算角色在撞墙前能够到达的最终格。
- 逐格播放平滑移动动画。
- 每到达一格便将显示位置校正到对应网格中心，避免累计浮点误差。
- 动画结束后通知流程控制器执行下一条指令。

### CommandQueue

- 接收四方向输入。
- 将指令数量限制为 X。
- 暴露是否已填满、清空和只读指令列表。
- 不负责角色移动或胜负判断。

### GameFlow

状态分为：

- `Input`：允许添加和清空指令。
- `Executing`：依次执行队列并锁定输入。
- `Succeeded`：显示成功并停止本关。
- `Failed`：显示失败，延迟后重置为 `Input`。

GameFlow 只在全部指令执行完后比较玩家坐标与终点坐标。

### GameUI

- 左下角显示四方向按钮。
- 底部显示 X 个指令槽及已输入的方向箭头。
- 右侧显示“执行”和“清空”按钮。
- 顶部显示输入数量提示、执行状态及成功或失败提示。
- 未填满 X 条时禁用“执行”；执行期间禁用所有操作按钮。

## 场景制作

每个关卡场景包含一个 `LevelRoot`，其下组织地板、障碍、起点、终点、玩家和 UI。

后续制作关卡时：

1. 复制现有关卡场景。
2. 修改地图宽度、高度和单格尺寸。
3. 在网格位置上移动、增加或删除障碍方块。
4. 调整起点和终点。
5. 手动配置 X。
6. 进入 Play Mode 验证。

障碍、起点和终点的世界位置必须对齐网格。启动验证失败时，在 Console 输出包含对象和错误原因的信息，并阻止关卡开始。

需要验证的错误包括：

- 地图宽度、高度或 X 小于 1。
- 玩家、起点或终点引用缺失。
- 起点或终点超出地图边界。
- 障碍超出地图边界。
- 起点或终点与障碍重叠。
- 同一格存在多个障碍。

## 输入

- `W` / 上方向键：添加向上指令。
- `S` / 下方向键：添加向下指令。
- `A` / 左方向键：添加向左指令。
- `D` / 右方向键：添加向右指令。
- 空格：队列填满后执行。
- `R`：输入阶段清空全部指令。
- 屏幕按钮提供与键盘相同的操作。

## 视觉表现

- 使用基础 3D 方块表现地板和障碍。
- 使用固定俯视斜角相机显示完整地图。
- 玩家、起点和终点使用容易区分的基础颜色。
- 角色移动使用逐格平滑动画，规则计算始终使用整数网格。
- 首版不依赖外部美术资源。

## 数据流

1. LevelController 验证场景并建立网格占用数据。
2. GameFlow 将角色设置到起点并进入 `Input`。
3. 输入被转换为方向并写入 CommandQueue。
4. 队列填满后，执行操作使 GameFlow 进入 `Executing`。
5. GameFlow 逐条读取指令，PlayerMover 计算目标格并播放移动。
6. 每条动画完成后执行下一条。
7. 队列结束后，GameFlow 检查最终坐标并进入 `Succeeded` 或 `Failed`。
8. 失败提示结束后重置角色和队列，返回 `Input`。

## 测试与验收

### 核心逻辑测试

- 少于 X 条时不能执行。
- 达到 X 条后不能继续添加。
- 清空后队列为空且可以重新输入。
- 角色沿方向移动到障碍前或边界前。
- 面前有障碍时角色不动且指令被消耗。
- 多条指令严格按顺序执行。
- 经过终点但最终未停在终点时失败。
- 全部指令完成且最终停在终点时成功。
- 执行期间忽略输入、清空和重复执行。
- 失败重置后角色、队列和状态均恢复初始值。

### 场景验收

- 鼠标按钮与两套键盘方向输入均可编排指令。
- 空格键与“执行”按钮行为一致。
- R 键与“清空”按钮行为一致。
- UI 指令槽与实际队列一致。
- 移动动画结束后角色准确对齐网格中心。
- 配置错误能够阻止运行并给出明确日志。

## 完成标准

在示例关卡中，玩家能够使用鼠标或键盘输入恰好 X 条指令，确认后观看角色逐条滑行；正确路线在最后一条结束时停在终点并成功，错误路线显示失败后完整重置，且所有核心规则通过测试。
